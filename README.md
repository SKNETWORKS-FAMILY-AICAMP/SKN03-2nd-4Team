## 1. 프로젝트 개요

**제주도는 연령대별 다양한 소비 패턴과 특정한 관광 목적을 가진 주요 관광지이다. 이 프로젝트는 제주도를 방문하는 내국인 및 외국인 관광객의 행동 데이터를 활용하여 개인화된 관광 일정을 제공함으로써 관광객 경험을 향상시키고, 지역 경제에 기여하는 것을 목표로 한다. 제공되는 데이터를 분석하여, 관광 트렌드를 파악하고, 이를 기반으로 관광객에게 최적화된 정보를 제공하는 시스템을 구축한다.**

## 4. 기술적 접근

### 4.1 데이터 전처리

### 4.1.1 데이터 수집

- **목적**: 프로젝트에 필요한 다양한 데이터 소스를 식별하고, 데이터를 수집하는 단계.
- **작업 내용**:
    - **데이터 소스 식별**: 공공 데이터 포털, 제주 관광 빅데이터 플랫폼, 기상청, 교통량 데이터 등에서 필요한 데이터를 식별.
    - **데이터 다운로드 및 API 연동**: 해당 소스에서 데이터를 다운로드, 혹은 API를 통해 실시간 데이터를 수집.
    - **데이터 저장**: 수집한 데이터를 raw 데이터 폴더에 저장.

### 4.1.2 데이터 클리닝

- **목적**: 수집된 데이터를 정리하고, 분석 및 모델링에 적합하도록 가공하는 단계이다.
- **작업 내용**:
    - **결측치 처리**:
        - **삭제**: 결측치가 많은 열이나 행을 삭제 혹은 null로 채운다
        - **대체**: 결측치를 평균값, 중앙값 또는 최빈값으로 대체한다.
    - **이상치 처리**:
        - **검출**: 박스 플롯(Box Plot) 등을 사용하여 데이터의 이상치를 검출한다.
        - **처리**: 검출된 이상치를 분석하여, 필요에 따라 값을 수정하거나, 데이터를 삭제한다.
    - **중복 데이터 처리**:
        - **중복 데이터 제거**
    - **형식 일치화**:
        - **날짜 및 시간 데이터**: 날짜 형식을 통일 및 필요에 따라 특정 시간대나 날짜 포맷으로 변환한다.
        - **텍스트 데이터**: 대소문자 통일, 불필요한 공백 제거 등

### 4.1.3 데이터 통합

- **목적**: 다양한 소스에서 수집된 데이터를 하나의 일관된 데이터셋으로 통합하는 과정이다.
- **작업 내용**:
    - **데이터 조인**:
        - **수평적 통합**: 공통된 키(예: 날짜, 위치)를 기반으로 여러 테이블이나 데이터셋을 조인한다.
        - **수직적 통합**: 유사한 구조를 가진 데이터셋을 합쳐, 단일 테이블로 구성한다.
    - **형식 통일**:
        - 데이터 소스 간의 형식 차이 조정 및 구조 통일.
    - **결합 후 클리닝**:
        - 통합 이후 데이터 정리 및 클리닝 작업을 수행한다.

### 4.1.4 피처 엔지니어링

- **목적**: 모델링에 필요한 추가 변수를 생성하거나, 기존 변수를 가공하여 모델의 성능을 향상시키는 단계.
- **작업 내용**:
    - **새로운 변수 생성**:
        - **파생 변수**: 기존 변수들을 조합하여 새로운 변수를 생성한다. 예를 들어, 날짜 변수에서 '요일', '주말 여부' 등을 추출하는 등.
    - **변수 변환**:
        - **로그 변환, 스케일링**: 데이터 분포를 정규화하거나, 특정 변수의 스케일을 맞추기 위해 로그 변환이나 스케일링을 수행.
    - **카테고리화**:
        - 연속형 변수를 카테고리 변수로 변환하여, 분석의 용이성을 높인다. 예를 들어, 연령 데이터를 '20대', '30대' 등으로 구분.

### 4.1.5 데이터 검증

- **목적**: 전처리된 데이터가 분석 및 모델링에 적합한지 검토하는 단계.
- **작업 내용**:
    - **기본 통계 확인**:
        - 평균, 중앙값, 분산 등을 확인하여 데이터의 특성을 파악.
    - **시각적 검토**:
        - 히스토그램, 상자 그림(Box Plot), 산점도 등을 활용하여 데이터 분포를 시각적으로 검토.
    - **샘플링 테스트**:
        - 작은 샘플 데이터를 활용하여 모델링 및 분석에 문제가 없는지 테스트.

### 4.2 지도 기반 시각화 기능 개발

### 4.2.1 요구 사항 분석

- **목적**: 사용자와 이해관계자의 요구를 분석하여 지도 기반 시각화 기능의 핵심 요소를 정의합니다.
- **작업 내용**:
    - **시각화할 데이터 항목 정의**: 관광지 위치, 날씨 정보, 교통량, 혼잡도, 관광객 분포 등 지도에 시각화할 주요 데이터를 정의합니다.
    - **사용자 시나리오 개발**: 사용자가 지도에서 어떤 정보를 얻고자 하는지, 어떻게 상호작용할지를 파악하여 다양한 사용자 시나리오를 작성합니다.
    - **성능 요구 사항 설정**: 지도 로딩 속도, 데이터 업데이트 주기, 상호작용의 반응 속도 등 성능에 대한 요구 사항을 설정합니다.

### 4.2.2 지도 API 선택 및 통합

- **목적**: 지도 기능을 제공하는 API를 선택하고, 이를 프로젝트에 통합하여 지도를 웹 플랫폼에 구현합니다.
- **작업 내용**:
    - **지도 API 선택**: Google Maps, Leaflet, Naver Maps 등 다양한 지도 API 중 프로젝트 요구 사항에 맞는 API를 선택합니다.
        - **Google Maps API**: 풍부한 기능과 광범위한 커스터마이징 옵션을 제공하며, 전 세계적으로 많이 사용됨.
        - **Leaflet**: 경량 오픈소스 라이브러리로, 커스터마이징과 확장성이 뛰어남.
        - **Naver Maps API**: 한국에서 제공하는 지도 서비스로, 국내 사용자에게 친숙한 인터페이스 제공.
    - **API 통합**:
        - 선택한 지도 API를 웹 애플리케이션에 통합합니다.
        - API 키 발급 및 설정, API 초기화, 기본 지도 설정(중심 위치, 줌 레벨 등) 등의 작업을 수행합니다.

### 4.2.3 데이터 시각화 설계 및 구현

- **목적**: 지도 위에 다양한 데이터를 시각적으로 표현하여, 사용자가 쉽게 이해할 수 있도록 합니다.
- **작업 내용**:
    - **관광지 위치 표시**:
        - 제주도 내 주요 관광지의 위치를 지도 위에 마커로 표시합니다.
        - **마커 스타일링**: 각 마커의 스타일(색상, 크기, 아이콘 등)을 설정하여, 관광지 유형(자연경관, 문화유산, 액티비티 등)에 따라 구분할 수 있도록 합니다.
        - **날씨 정보 시각화**:
        - 실시간 날씨 데이터를 기반으로 지도 위에 날씨 정보를 표시합니다. 예를 들어, 특정 지역에 비가 오는 경우 해당 지역에 비 아이콘을 표시합니다.
        - **날씨 레이어**: 날씨 정보를 별도의 레이어로 설정하여, 사용자가 원하는 경우에만 활성화할 수 있도록 합니다.
        - **교통량 및 혼잡도 시각화**:
        - 제주도 내 주요 도로의 교통량 데이터를 기반으로, 혼잡도를 색상 또는 굵기로 표현합니다.
        - **혼잡도 히트맵**: 특정 시간대나 날짜의 혼잡도를 히트맵으로 표현하여, 사용자가 혼잡한 지역을 쉽게 파악할 수 있도록 합니다.
        - **관광객 분포도 시각화**:
        - 관광객의 실시간 위치 데이터를 기반으로, 제주도 내 관광객 분포를 시각화합니다.
        - **클러스터링**: 관광객이 밀집한 지역을 클러스터링 기법을 사용하여 시각화하고, 해당 지역의 혼잡도를 직관적으로 표현합니다.

### 4.2.4 인터랙티브 기능 구현

- **목적**: 사용자가 지도와 상호작용하며 필요한 정보를 직관적으로 얻을 수 있도록 인터랙티브 기능을 구현합니다.
- **작업 내용**:
    - **마커 클릭 및 팝업**:
        - 사용자가 지도 위의 마커를 클릭하면, 해당 관광지에 대한 상세 정보(예: 관광지 설명, 현재 날씨, 혼잡도 등)를 팝업 창으로 표시합니다.
        - **사진 및 리뷰 표시**: 팝업 창에 관광지의 사진, 사용자 리뷰 등의 추가 정보를 포함하여, 사용자가 더 많은 정보를 얻을 수 있도록 합니다.
    - **레이어 토글**:
        - 사용자가 지도 위의 다양한 정보를 선택적으로 볼 수 있도록, 날씨, 교통량, 관광객 분포 등의 레이어를 켜고 끌 수 있는 기능을 제공합니다.
        - **레이어 관리 패널**: 지도 상단 또는 측면에 레이어 관리 패널을 배치하여, 사용자가 쉽게 접근할 수 있도록 합니다.
    - **필터링 기능**:
        - 사용자가 특정 조건(예: 특정 유형의 관광지, 혼잡도 낮은 지역 등)에 따라 검색하고, 필터링된 결과를 지도에 시각화할 수 있도록 지원합니다.
        - **검색 연동**: 사용자가 검색창에 입력한 키워드와 필터링 옵션을 지도와 연동하여, 관련 결과만을 표시합니다.

### 4.2.5 성능 최적화

- **목적**: 지도 기반 시각화 기능이 대규모 데이터에서도 원활하게 작동하고, 사용자에게 빠른 응답성을 제공할 수 있도록 최적화합니다.
- **작업 내용**:
    - **데이터 캐싱**:
        - 자주 사용되는 데이터(예: 관광지 위치, 기본 지도 레이어 등)를 캐싱하여, 로딩 속도를 향상시킵니다.
    - **데이터 압축**:
        - 전송되는 데이터의 크기를 줄이기 위해 데이터를 압축하여 전송합니다. 예를 들어, GeoJSON 데이터의 크기를 줄여 전송 속도를 개선합니다.
    - **비동기 데이터 로딩**:
        - 데이터를 비동기적으로 로딩하여, 사용자가 지도를 탐색하는 동안 필요한 데이터만 로드하고, 전체 지도 성능을 저해하지 않도록 합니다.

### 4.2.6 테스트 및 사용자 피드백 반영

- **목적**: 구현된 지도 기반 시각화 기능이 요구 사항을 충족하고, 사용자가 기대하는 수준의 성능과 사용자 경험을 제공하는지 검증합니다.
- **작업 내용**:
    - **기능 테스트**:
        - 각 시각화 기능(마커 표시, 팝업, 레이어 토글 등)에 대해 개별 테스트 진행.
    - **성능 테스트**:
        - 로딩 시간과 반응 속도 측정.
    - **개선 사항 반영**: 수집된 피드백을 바탕으로 지도 기능 개선 및 조정.

### 5.3 시스템 구조

### 5.3.1 개발 환경

- **프로그래밍 언어**: Python
- **백엔드 프레임워크**: Django
- **데이터베이스**: MySQL
- **클라우드 인프라**: AWS (EC2, RDS, S3)

### 5.3.2 아키텍처 설계

- **클라이언트-서버 모델**:
    - **클라이언트**: 웹 브라우저(Chrome, Firefox, Safari, Edge 등)에서 HTML, CSS, JavaScript 기반의 프론트엔드 애플리케이션을 실행.
    - **서버**: Django 기반의 백엔드 애플리케이션이 클라이언트 요청을 처리하고, 동적 HTML 페이지를 렌더링하여 응답.
- **Nginx와 Gunicorn**:
    - **Nginx**: HTTP & 리버스 프록시 서버로, 클라이언트의 요청을 Gunicorn에 전달하고, 응답을 클라이언트에 반환.
    - **Gunicorn**: WSGI 서버로, Nginx로부터 받은 요청을 Django 애플리케이션에 전달하고 결과를 반환.
- **AWS 인프라**:
    - **EC2 인스턴스**: Django와 Nginx, Gunicorn 서버를 운영하는 인프라.
    - **RDS**: MySQL 데이터베이스 관리.
    - **S3**: 정적 파일 및 미디어 파일을 저장하고 제공하는 스토리지 서비스.

### 5.4 데이터베이스 설계

### 5.4.1 ERD (Entity Relationship Diagram)

- **엔티티**: 관광지, 방문자, 소비 기록, 기상 데이터 등.
- **관계**: 각 엔티티 간의 관계 정의 (예: 방문자와 소비 기록 간의 1:N 관계 등).

### 5.4.2 테이블 설계

- **필드**: 각 테이블의 필드명, 데이터 유형, 제약 조건 (예: `VARCHAR`, `INT`, `PRIMARY KEY` 등).
- **인덱스**: 검색 성능을 향상시키기 위한 인덱스 설정.

### 5.5 UI 설계

### 5.5.1 메인 페이지

- **구성 요소**: 지도 시각화, 주요 관광지 표시.

### 5.5.2 지도 시각화 페이지

- **기능**: 사용자가 지도에서 관광지와 실시간 정보를 탐색할 수 있는 인터페이스.

### 5.5.3 상세 정보 페이지

- **구성 요소**: 선택한 관광지의 상세 정보, 리뷰, 관련 통계 데이터 표시.
